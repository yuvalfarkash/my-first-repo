# Quick Reference

📂

git init

Create a new Git repository in your current directory. Adds a hidden .git folder.

📷

git commit

Save a snapshot of staged changes with a message using -m "message".

🔌

git branch

Create, list, or delete branches. Work on features without affecting main.

☁️

git push

Upload local commits to a remote repository like GitHub.

🔃

git pull

Fetch and merge remote changes into your local branch in one step.

🔧

git merge

Combine changes from one branch into another. Usually into main.

# Initial Setup

Configure Git with your identity before making any commits.

Terminal Config 
# Set your name and email (used in every commit)
$ git config --global user.name "Your Name"
$ git config --global user.email "you@example.com"

# Set default branch name to 'main'
$ git config --global init.defaultBranch main

# Verify your configuration
$ git config --list
user.name=Your Name
user.email=you@example.com
init.defaultbranch=main

# Basic Commands

The core workflow: init, add, commit, status, log.

Terminal Essential 
# Initialize a new repository
$ git init
Initialized empty Git repository in /home/user/my-project/.git/

# Check the status of your files
$ git status
On branch main
Untracked files:
  (use "git add <file>..." to include in what will be committed)
        index.html
        style.css

# Stage specific files for commit
$ git add index.html style.css

# Or stage ALL changes at once
$ git add .

# Commit with a descriptive message
$ git commit -m "Add homepage layout and styles"
[main (root-commit) a1b2c3d] Add homepage layout and styles
 2 files changed, 85 insertions(+)

# View commit history
$ git log --oneline
a1b2c3d Add homepage layout and styles
Tip

Use git status often! It tells you which files are staged, modified, or untracked.

# Branching & Merging

Branches let you work on features in isolation, then merge them back.

Terminal Branches 
# List all branches (* marks current branch)
$ git branch
* main

# Create a new branch
$ git branch feature-navbar

# Switch to the new branch
$ git checkout feature-navbar
Switched to branch 'feature-navbar'

# Shortcut: create AND switch in one command
$ git checkout -b feature-footer
Switched to a new branch 'feature-footer'

# Do your work, add and commit...
$ git add .
$ git commit -m "Add footer component"

# Switch back to main and merge
$ git checkout main
$ git merge feature-footer
Updating a1b2c3d..e4f5g6h
Fast-forward
 footer.html | 25 +++++++++++++++++++++++++
 1 file changed, 25 insertions(+)

# Delete the branch after merging
$ git branch -d feature-footer
Deleted branch feature-footer (was e4f5g6h).

# Working with GitHub

Connect your local repo to GitHub and sync changes.

Terminal Remote 
# Connect to a remote repository
$ git remote add origin https://github.com/username/my-project.git

# Push your code to GitHub (first time)
$ git push -u origin main
Enumerating objects: 6, done.
Counting objects: 100% (6/6), done.
To https://github.com/username/my-project.git
 * [new branch]      main -> main
Branch 'main' set up to track remote branch 'main' from 'origin'.

# After first push, just use:
$ git push

# Pull latest changes from GitHub
$ git pull origin main

# Clone an existing repository
$ git clone https://github.com/username/my-project.git
Cloning into 'my-project'...
remote: Enumerating objects: 42, done.
remote: Total 42 (delta 0), reused 0 (delta 0)
Receiving objects: 100% (42/42), done.
Warning

Never push sensitive data (passwords, API keys, .env files) to GitHub. Add them to your .gitignore file first!

# Undoing Changes

Made a mistake? Git has several ways to fix it.

Terminal Undo 
# Unstage a file (keep the changes, remove from staging)
$ git reset HEAD index.html

# Discard changes in a file (revert to last commit)
$ git checkout -- index.html

# Undo the last commit but KEEP changes staged
$ git reset --soft HEAD~1

# Undo the last commit and UNSTAGE changes
$ git reset HEAD~1

# View what changed in a file
$ git diff index.html
- <h1>Old Title</h1>
+ <h1>New Title</h1>

# See who changed each line of a file
$ git blame index.html
